  1. Head: Sirve para colocar el contenido principal de la página.
  2. Body: Dentro de esta etiqueta se suele colocar todo el contenido de la página.
  3. p: Crea saltos de línea para separar el texto que escribamos.
  4. Hn (n=1,2,3,4): Sirve para poner encabezados, se ve muy bien desde mi punto de vista. :)
  5. img: Sirve para colocar imágenes, por ejemplo, esa es mi fursona.
  6. hr: crea una línea horizontal por toda la página

Preguntas y respuestas:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    2. El internet consiste de una red gigantesca de computadoras que están conectadas por todo el mundo, de hecho existió antes que la WWW. La World Wide Web se compone de todas las páginas que existen, es una parte de todo Internet.

  2. ¿Cuáles son las partes de un URL?
    3. Protocolo, dominio, ruta y parámetros.

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    4. GET sirve para pedir datos de un recurso en específico

    > POST sirve como método de envío de datos a un servidor para crear un recurso

    > PUT sirve para actualizar el recurso existente en un servidor utilizando el contenido del cuerpo de la solicitud.

    > HEAD sirve para regresar un recurso que no contenga el cuerpo al momento de obtener la respuesta.

    > TRACE sirve como método de diagnóstico, extrae lo que está exactamente en el contenido del cuerpo.

    > OPTIONS sirve para describir las opciones de comunicación http disponibles para el recurso de destino.

    > PATCH sirve para aplicar modificaciones parciales a un recurso.

    > DELETE sirve para eliminar un recurso específico.

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    5. POST, porque sirve para enviar datos al servidor.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    6. GET, porque así se piden los datos al servidor.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    7. Que todo está bien, un request satisfactorio.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    8. No siempre, el usuario pudo haber cometido un error. Si debería haber algo ahí, sí debe reportarse.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    9. Sí, porque es un error del servidor directamente.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    10. Que ya no será soportado pronto y que su uso ya no es recomendable por posible incompatibilidad.

    11. HTML4 es más desordenado que HTML5, HTML5 incluye mejor soporte para más dispositivos y páginas más intuitivas en cuanto a diseño.

  • El cliclo de vida y desarrollo de los sistemas de información:
    1. ¿Cuál es el ciclo de vida de los sistemas de información?
      2. > Planificación: Se prepara el diseño e implementación del sistema

      > Fase de análisis: Se analizan y se concretan los requisitos del sistema, así como sus cambios si se trata de una actualización.

      > Fase de diseño: Se determina el cómo el sistema cumplirá con los requisitos, con soluciones potenciales que dependen de costes, flexibilidad y efectividad.

      > Fase de desarrollo: Desarrollar lo que está planeado de acuerdo con las fases anteriores.

      > Fase de integración y periodo de pruebas: Verificar que el diseño cumple con los requisitos, revisión de errores y prueba de usuarios para ajustarlo a que sea más fácil e intuitivo para quienes lo usarán.

      > Fase de implementación: Adquirir e instalar hardware y software de acuerdo a los requisitos y probar nuevamente el sistema.

      > Fase de mantenimiento: Ajustes necesarios para mejorar y mantener operable el sistema mediante respaldos, soporte al usuario, resolver fallos y optimizar el sistema en velocidad o seguridad.

    2. ¿Cuál es el ciclo de desarrollo de sistemas de información?
      3. Es un ciclo bastante similar al de vida, cambia en algunos procesos según lo que he podido investigar en otros lados:

      > Identificación de problemas, oportunidades y objetivos.

      > Determinación de los requerimientos de información.

      > Análisis de las necesidades del sistema.

      > Diseño del sistema recomendado.

      > Desarrollo y documentación del software.

      > Pruebas y mantenimiento del sistema.

      > Implementación y evaluación del sistema.
